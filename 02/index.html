web문서의 주의 사항

브라우저에서 해석하고 출력할 수 있는 모든 문서를 의미함

  • 시멘틱 마크업을 지향한다.
  • 웹표준 : w3c?(html,css,java등 웹의 표준을 만드는곳 이므로 제시한 문법을 준수
  • 웹접근성 : 모든사람이 모든환경에서 웹을 사용할수 있도록 웹 문서를 제작해야 함 (aria,role)
  • 크로스 브라우징(브라우저 호환성):5대브라우저에서 동일한 결과화면과 내용을 출력할수 있어야 함
    • 5대 브라우저:(IE,chrome,safari,firefox,opera)앱 브라우저:(chrome,safari)

html언어의 개념

웹 문서의 기본구조를 구축하는 언어임
(Hyper:바로 연결되는 Text:글자로만 Makeup:요소를 정리하다 Language:언어)의 약어가 HTML

특징과 주의사항들

  • 웹문서의 구조(DOM)는 요소로 구성되어있다.
  • 요소(element)는 태그(tagname)와 속성(attribute)로 구성된다.
  • 요소간의 노드(Node)가 존재한다
    부모(parent),자식(children),자손(decendant),형제(siblings)
  • 노드의 구조는 정해져 있다.

다양한 시멘틱 요소

  • h1~h6:제목을 의미함
    모든웹문서에는 h1을 하나이상 포함해야함(한개만 사용할것을 권장함) h1~h6까지 단계적으로 사용할 것을 권장함
  • p태그:관련된 하나의 문장(구,절,단어두개이상) 범위를 의미함
  • ul,ol,li:목록을 의미함
  • strong,em,b태그 :강조,중요함을 의미함(가급적 strong태그를 쓸것)
  • i태그:구분의 역할(시각적 효과로 폰트가 기울어짐을 볼 수 있다.)
  • small:보조적 의미
  • adress:주소,연락처,이름
  • blockquote,q 태그:인용
  • kbd:키보드
  • time:시간
  • div,span: 의미가 없는 의미
